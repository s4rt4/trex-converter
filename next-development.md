# T-Rex Converter — Next Development Roadmap

Roadmap fitur lanjutan untuk memaksimalkan kemampuan engine yang sudah dipakai (ImageMagick, FFmpeg, LibreOffice, PyMuPDF, Tesseract, qpdf) ditambah usulan modul baru.

Format checklist: `[ ]` belum, `[x]` selesai, `[~]` sebagian.

Setiap task yang selesai ditandai `[x]` dan diberi catatan tanggal/commit pendek bila relevan.

---

## 1. Image Module (ImageMagick)

Tujuan: ekspos kemampuan ImageMagick di luar resize/quality/strip dasar.

- [x] Engine: terjemahkan opsi lanjutan ke argumen ImageMagick (transform, color, filter, watermark, border)
- [x] Engine: ICO multi-resolution otomatis via `-define icon:auto-resize=...`
- [x] UI: `ImageOptionsPanel` dengan section Transform / Color / Filter / Watermark / Output
- [x] Transform: rotate, flip, flop, auto-trim, free-crop `WxH+X+Y`, crop aspect (`1:1`, `4:5`, `16:9`, `3:2`)
- [x] Resize mode: dimension langsung, longest-edge, percent, megapixel target
- [x] Color: grayscale, sepia, negate, normalize, brightness, contrast, gamma
- [x] Filter: blur, sharpen, denoise (-enhance), vignette
- [x] Border & frame: ukuran + warna border
- [x] Watermark teks: konten, posisi (gravity), opacity
- [x] Density (DPI) untuk output raster
- [x] Watermark gambar (PNG overlay) dengan resize & opacity — Image page Watermark tab dapat field "Image" + "Image width", `_image_watermark_args` build paren-block `( WATERMARK_FILE -resize WxH -alpha set -channel A -evaluate multiply ALPHA +channel )` lalu `-gravity G -geometry +X+Y -compose over -composite`.
- [x] Smart fit-to-canvas dengan letterbox + warna background — `fit_canvas` option emit `-resize WxH -background COLOR -gravity center -extent WxH`. Image options Transform tab dapat field "Fit canvas" + "Canvas bg".
- [ ] Color profile: convert ke sRGB, embed/strip ICC
- [ ] Animated GIF/WebP optimize: dedupe frame, set loop, extract frame ke-N
- [x] Montage: grid kolase dari N gambar — Image Montage page (multi_input + force_engine + default_options=operation:montage) → ImageMagick `montage` subcommand dengan tile (auto sqrt-ish atau eksplisit `3x3`), geometry, background color.
- [ ] EXIF granular: strip GPS only, baca/edit tag tertentu (butuh exiftool atau PyExifTool)
- [ ] Batch drag-and-drop banyak gambar sekaligus

## 2. Video Module (FFmpeg)

- [x] Trim/cut by timestamp (start/end), copy codec kalau bisa — trim re-encode default; opt-in `stream_copy` checkbox emit `-c copy` dan skip filter chain (filter/CRF di-ignore saat stream-copy aktif).
- [x] Concat multi-file — Video Concat page (multi_input + force_engine + default_options=operation:concat) → FFmpeg `concat` filter complex (re-encode, v=1:a=1 untuk video, v=0:a=1 untuk audio output).
- [x] Compress dengan target ukuran (two-pass) atau target CRF — CRF + preset selesai; two-pass selesai via opsi `target_size_mb` yang trigger flow `_run_two_pass` (probe duration via ffprobe, hitung video bitrate = (target_kbps - audio_kbps), jalankan `-pass 1 -an -f null /dev/null` lalu `-pass 2 -b:v Xk`).
- [x] Resolution preset 4K → 1080p / 720p / 480p (plus 1440p, 360p)
- [x] GIF / WebP creator dari clip video (palette gen untuk GIF) — palettegen + paletteuse untuk GIF, libwebp + loop=0 untuk WebP animasi
- [x] Thumbnail / contact sheet (N frame jadi grid) — `select=not(mod(n,N))` + `tile=ColsxRows` ke PNG/JPG
- [x] Watermark logo overlay (posisi, waktu mulai, opacity) — overlay via `-filter_complex` dengan gravity 9-arah, scale, dan opacity (`colorchannelmixer=aa`); watermark teks tetap tersedia
- [x] Subtitle burn-in (SRT/ASS hardcode) — `subtitles=` filter (.srt/.vtt) atau `ass=` filter (.ass/.ssa) di video filter chain
- [x] Subtitle extract dari MKV — pairs `(mkv|mp4|mov|webm, srt|ass|vtt)`. Engine emit `-map 0:s:N -c:s codec` (codec map: srt→srt, ass→ass, vtt→webvtt). Sidebar entri "Subtitle Extract" (force_engine=ffmpeg, kind=subtitle-extract).
- [x] Hardware acceleration detect (vaapi / nvenc / qsv) — helper `detect_hardware_accels()` (async via `ffmpeg -hide_banner -hwaccels`) dan `_detect_hwaccels_sync()` (subprocess+timeout). Hasil append ke dependency dialog sebagai "FFmpeg hardware accel: vaapi, cuda, ...".
- [x] Speed change (slowmo / timelapse) — 0.5x–2.0x via setpts + atempo
- [x] Rotate / flip / crop video
- [x] Reverse video — `reverse` + `areverse` filter (terakhir di chain)

## 5. PDF Tools (PyMuPDF + qpdf)

ROI tertinggi karena tool gratis dan use-case-nya luas.

- [x] Merge beberapa PDF jadi satu — POC pertama dari refactor multi-input Task model. PDF Merge sidebar page (multi_input + force_engine + default_options=operation:merge) feed `task.inputs` ke `PDFEngine._run_merge` yang iterate `Document.insert_pdf` per source.
- [x] Split by range / by N-pages / by ukuran file — semua tiga mode selesai. `split_mode=size` + `split_size_mb` trigger `_run_split_by_size` yang iteratif: insert page, serialisasi via `Document.tobytes(garbage=3, deflate=True)`, kalau size > limit AND chunk punya >1 page maka rollback (delete_page) dan finalize chunk sebelumnya, mulai chunk baru dengan page yang overflow.
- [x] Extract pages dengan range syntax (`1-3,5,8-10`)
- [x] Reorder & rotate halaman
- [x] Compress (PyMuPDF garbage + deflate + clean) — image downsample dan linearize via qpdf SELESAI. Operation `compress_images`: per halaman iterate `Page.get_images(full=True)`, dedupe by xref, hitung effective DPI (pixel dim ÷ rendered size in inches), kalau > target_dpi shrink via `fitz.Pixmap(source, new_w, new_h)` lalu `tobytes("jpeg" or "png")` (PNG kalau punya alpha) dan `Document.update_stream(xref, new_bytes)`. Operation `linearize`: `qpdf --linearize input output`.
- [x] Encrypt / decrypt (password owner & user) — AES-256
- [x] Watermark teks atau gambar — keduanya selesai. Image watermark via PyMuPDF `page.insert_image(rect, filename, overlay=True)`, posisi via gravity 9-arah, lebar via `watermark_image_width_fraction` (% halaman), opacity via overlay `draw_rect(fill_opacity=...)`.
- [x] Page numbering / Bates numbering — `_op_page_numbering` di PDFEngine, template format string dengan `{n}`/`{total}`/`{page}`, gravity 9-arah, font size, start number, skip-first-N, opacity. Sidebar entri "PDF Numbering" (single-input → pdf).
- [x] Metadata edit (title/author/subject/keywords) atau strip — keduanya selesai (action combo Strip / Edit)
- [x] Repair PDF korup (qpdf) — `qpdf input output` round-trip; treats exit 3 (warnings) as success
- [x] PDF → DOCX — selesai via `pdf2docx.Converter(src).convert(dst)` di `asyncio.to_thread`. Pair `(pdf, docx)` ditambahkan ke SUPPORTED_PAIRS.
- [x] PDF → HTML / TXT / EPUB (PyMuPDF native) — TXT/HTML/EPUB selesai. EPUB builder `_write_epub` di `pdf_engine.py`: minimal EPUB 2 (mimetype STORED + META-INF/container.xml + OEBPS/content.opf + toc.ncx + chapter-NNNN.xhtml per page). Body XHTML dari `page.get_text("html")` di-strip outer `<html><body>` dan re-wrapped dalam scaffold XHTML 1.1.
- [x] Extract embedded images — pdf → folder dispatch via `options.operation=extract_images`, iterate `Page.get_images(full=True)` per halaman, dedupe by xref, `Document.extract_image(xref)` tulis `<stem>-pageN-imgM.<ext>`. Sidebar entri "PDF Extract Images".
- [x] Extract embedded attachments — pdf → folder dispatch via `options.operation=extract_attachments`, walk `Document.embfile_count`/`embfile_info`/`embfile_get`, sanitize nama via `_safe_attachment_name`. Sidebar entri "PDF Extract Attachments".
- [x] Redaction (PyMuPDF `add_redact_annot`) — operasi `redact` di PDFEngine: per halaman `Page.search_for(term)` untuk setiap term (comma-separated), `add_redact_annot(rect, fill=color)` lalu `apply_redactions()` (glyphs dihapus, bukan ditutup). PDFOperationsPanel dapat tab "Redact" (search terms / color preset / pages filter).

## 9. SVG / Vector Tools (Inkscape) — modul baru

Tujuan: kemampuan vector-grade conversion dan SVG editing yang ImageMagick / PyMuPDF tidak cover. ImageMagick rasterize SVG via librsvg fallback (gradient/filter/text sering rusak); Inkscape adalah gold-standard untuk SVG.

### Wave 1 — rendering & cleanup (paling sering dibutuhkan)

- [x] SVG → PNG dengan DPI / width / height — `InkscapeEngine` build `inkscape --export-type=png --export-filename=out.png [--export-dpi=N] [--export-width=W] [--export-height=H] in.svg`. Width/Height override DPI per Inkscape semantics.
- [x] SVG → PDF (vector preserved) — `--export-type=pdf` dengan filename + input path saja.
- [x] SVG → SVG **trim-to-content** — operation `trim` emit `--export-type=svg --export-plain-svg --vacuum-defs --export-area-drawing`; viewBox di-crop ke bounding box drawing.
- [x] SVG → SVG **plain cleanup** — operation `cleanup` (default) emit `--export-type=svg --export-plain-svg --vacuum-defs`; drop namespace `inkscape:`/`sodipodi:`, hapus `<defs>` yang tidak terpakai.

### Wave 2 — vector interop

- [x] SVG → EPS / PS (PostScript variants untuk print pipeline) — `--export-type=eps|ps` plus opsional `--export-ps-level=2|3` dan `--export-text-to-path`. Default level = Inkscape default (3).
- [x] SVG → EMF / WMF (Windows print / Office embed) — `--export-type=emf|wmf`. Tidak ada opsi text-to-path (EMF/WMF tidak support).
- [x] PDF → SVG dengan vektor dipertahankan — pair `(pdf, svg)` di-route ke InkscapeEngine via registry; emit `--export-type=svg --export-plain-svg [--pages=N]` (default page 1). Multi-page → folder belum (Wave 2.5).
- [x] **Text → paths** (`--export-text-to-path`) — option `text_to_path: bool` di SVGOptionsPanel (checkbox), engine emit flag untuk pdf/eps/ps/svg output (EMF/WMF skip karena format tidak support).

### Wave 3 — utilities & advanced

- [x] Export single object/layer by ID — option `inkscape_export_id` (str) emit `--export-id=ID`; option `inkscape_export_id_only` (bool) emit tambahan `--export-id-only`. Berlaku untuk semua output (png/pdf/svg/eps/ps/emf/wmf), tidak diaplikasikan pada pdf→svg input direction (irrelevant). UI: text field "Export ID" + checkbox "Hide other objects" di SVGOptionsPanel. Builder skip flag bila id-only set tanpa id.
- [x] DXF round-trip (CAD interop) — pair `(svg, dxf)` via `--export-extension=org.ekips.output.dxf_outlines` (R14 default) atau `org.inkscape.output.dxf_twelve` (R12) lewat option `inkscape_dxf_format`; pair `(dxf, svg)` via `--export-type=svg --export-plain-svg`. Catatan: extension-based export tidak pakai `--export-type` flag (extension menentukan format output). Butuh `python3-tinycss2` apt package (Inkscape extension dxf_outlines.py mengimpor inkex yang depends pada tinycss2).
- [x] Pixmap → SVG trace (Inkscape Trace Bitmap) — pair `(png|jpg|jpeg|bmp|tif|tiff|gif|webp, svg)` via pipeline `magick INPUT -colorspace Gray TEMP.pgm` lalu `potrace TEMP.pgm -s -o OUTPUT.svg`. Options: `trace_threshold` (0.0–1.0, potrace `-k`), `trace_turdsize` (potrace `-t`), `trace_alphamax` (potrace `-a`). Engine declare `extra_binaries=("potrace", "magick")`. SVG page input formats diperluas dengan 8 bitmap formats + dxf.

### Catatan dependency

- Inkscape ~150MB di Debian (`apt install inkscape`). Worth-it kalau Wave 1 + 2 minimum dikerjakan sekaligus (≥6 fitur).
- Alternatif **lightweight** kalau cuma butuh sebagian:
  - `librsvg2-bin` / `rsvg-convert` (~5MB): cover SVG → PNG/PDF/PS/EPS rendering, **tapi tidak bisa edit** (no trim, no cleanup, no text-to-path, no SVG-out).
  - Python-only trim: `lxml` parse + bbox computation per element type (`<rect>/<circle>/<path d=…>`), update `viewBox`. ~150 LOC, gratis, tapi rapuh untuk SVG kompleks (filter, clipPath, `<use>`, text dengan font metrics).

### Saran sequencing

Wave 1 paling tinggi value-per-effort: render berkualitas + trim + cleanup. Wave 2 nice-to-have untuk print/Office crowd. Wave 3 niche, bisa skip kecuali ada permintaan.
