# Changelog

## Unreleased

- SVG / Vector Wave 2: vector interop. `InkscapeEngine` adds five new pairs — `svg→eps`, `svg→ps`, `svg→emf`, `svg→wmf`, and `pdf→svg`. EPS/PS output respects `inkscape_ps_level` (2 or 3, validated) and the new `text_to_path` boolean (emits `--export-text-to-path` for PDF/EPS/PS/SVG output; EMF/WMF skip it because the formats don't support outlined text). PDF→SVG accepts `inkscape_pdf_page` (1-indexed, validated) and emits `--pages=N --export-plain-svg`. The SVG sidebar page now accepts PDF input alongside SVG; `SVGOptionsPanel` grows a PS-level combo, a PDF-page spinbox, and a Text→paths checkbox. `ConversionPage._populate_outputs` was tightened so that, when `force_engine=True`, output choices are additionally filtered through the forced engine's `supports(format_in, output)` — that prevents the PDF→PNG combination from appearing on the SVG page (where it would otherwise route to Inkscape via force-engine and fail) and applies the same safeguard to every other forced-engine page.
- New SVG / Vector sidebar page powered by `InkscapeEngine` wrapping `inkscape` 1.4 (Wave 1 of the SVG roadmap). Pairs: `svg→png` (`--export-type=png` with optional `--export-dpi`, `--export-width`, `--export-height`), `svg→pdf` (`--export-type=pdf`, vector-preserved), and `svg→svg` with two operations: `cleanup` (default — `--export-plain-svg --vacuum-defs`) and `trim` (cleanup + `--export-area-drawing` to crop the viewBox to the drawing's bounding box). New `SVGOptionsPanel` exposes Raster DPI / Width / Height spinboxes (with "auto" as the special zero-value) and an SVG operation combo. Engine cancels via subprocess terminate/kill, and a post-run `output_path.exists()` check converts silent-no-output runs into a clear error.
- Batch 7 — cross-module polish (4 fitur). **Archive compress**: ArchiveEngine grows a `folder → zip/tar/tgz/tbz/txz` direction (stdlib `zipfile.ZIP_DEFLATED` + `tarfile.open(mode=w/w:gz/w:bz2/w:xz)`), iterates `Path.rglob` and preserves relative POSIX arcnames. New `directory_input` flag on `ConversionPageConfig` (Browse opens `QFileDialog.getExistingDirectory`, `format_in` derives from config rather than empty `Path.suffix`). Sidebar entri "Archive Compress". **Password-protected PDF export**: LibreOfficeEngine `_convert_to_format` now emits a JSON-encoded `pdf:writer_pdf_Export:{...}` filter combining `pdf_a`, `pdf_password_user` (DocumentOpenPassword + EncryptFile), and `pdf_password_owner` (PermissionPassword + RestrictPermissions). DocumentOptionsPanel grows two QLineEdit password fields (Echo Password mode). **PDF redaction**: PDFEngine `redact` operation iterates `Page.search_for(term)` per term, calls `Page.add_redact_annot(rect, fill=color)` then `Page.apply_redactions()` so the original glyphs are removed (not just covered). PDFOperationsPanel gets a new "Redact" tab (terms / color / page range). **Image fit-to-canvas**: ImageMagick gains `fit_canvas` option emitting `-resize WxH -background COLOR -gravity center -extent WxH` (overrides regular `-resize` to avoid double-scaling). ImageOptionsPanel Transform tab adds Fit canvas + Canvas bg fields.
- Batch 6 — PDF & Document expansion. **PDF page numbering** (Bates-style): new `page_numbering` operation in PDFEngine writes a configurable template (`{n}`/`{total}`/`{page}` placeholders) onto each page via `Page.insert_text`, with 9-position gravity, font size, start number, skip-first-N pages, and opacity. **PDF extract images** (pdf→folder, `extract_images`): iterates pages, dedupes by xref, extracts via `Document.extract_image(xref)`, writes `<stem>-page<P>-img<I>.<ext>`. **PDF extract attachments** (pdf→folder, `extract_attachments`): walks `embfile_count`/`embfile_info`/`embfile_get`, dumps each embedded file with sanitized name. **PDF/A archival output**: `DocumentOptionsPanel` exposes a "PDF/A-1a archival" checkbox; LibreOfficeEngine routes that flag to the writer filter `pdf:writer_pdf_Export:{"SelectPdfVersion":{"type":"long","value":"1"}}`. **Slides → images** (pptx/odp/etc → folder): LibreOffice converts to PDF in a tempdir, then PyMuPDF Pixmap renders each page at configurable DPI to PNG/JPG as `<stem>-001.<ext>`. Four new sidebar pages wire these up through `force_engine`/`default_options`/`directory_output`. Sidebar scrollbar shrunk to 6px with a soft semi-transparent handle.
- PDF Split sidebar page (single input, directory output): new PDFEngine `split` operation supports `every_n` (with `split_pages_per_file`) or `range` (with `split_ranges` like `1-5, 6-10`). Outputs land in the chosen folder as `<stem>-001.pdf`, `<stem>-002.pdf`, ...
- PDF watermark with images: PDFEngine adds a `watermark_image` operation that places an image on every page via PyMuPDF `Page.insert_image`, with 9-position gravity, configurable page-width fraction (default 25%), and translucent fill emulating opacity. PDF Tools Watermark tab gains image picker and image-width input; the engine routes to `watermark_image` when an image path is set, falling back to `watermark_text` otherwise.
- Image watermark PNG overlay: ImageMagickEngine builds a paren-block `( WATERMARK_FILE -resize WxH -alpha set -channel A -evaluate multiply A +channel ) -gravity G -geometry +X+Y -compose over -composite`. Image options Watermark tab gains image picker and image width input alongside the existing text watermark fields.
- Audio cover art (ID3 APIC): FFmpegEngine attaches `cover_art_path` as a second input with `-map 0:a -map 1 -c:v copy -disposition:v attached_pic -metadata:s:v title='Album cover' -id3v2_version 3` whenever the output is audio. Audio options Tags tab gains a cover-art picker; existing audio filters and ID3 metadata still apply.
- New Document Merge sidebar page: LibreOfficeEngine grows a `_bulk_merge_to_pdf` operation that converts each non-PDF input to PDF in a tempdir (PDF inputs pass through) and concatenates them via PyMuPDF `Document.insert_pdf`. Single-shot conversion logic was refactored into a reusable `_convert_one(task, input_path, output_dir, format_out)` helper.
- Multi-input wave: four new sidebar pages built on the multi-input foundation. **Video Concat** routes mp4/mov/mkv/webm × N → mp4/mov/mkv/webm via FFmpeg `concat` filter complex (re-encodes; emits `[0:v:0][0:a:0]…concat=n=N:v=1:a=1[outv][outa]` plus matching `-map`s). **Audio Mix** routes audio × N → audio via FFmpeg `amix` filter with `duration` (longest/shortest/first) and a normalize toggle. **Image Montage** routes image × N → image via the ImageMagick `montage` subcommand (auto sqrt-ish tile or explicit `WxH`, validated geometry, configurable background). **Subtitle Merge** concatenates SRT/VTT/ASS files in any combination via Python with mode `shift` (cumulative offset + gap, default) or `append` (sort by start). All four pages run with `multi_input=True`, `force_engine=True`, and `default_options` injecting the appropriate operation key.
- Multi-input refactor on the core `Task` model: added `extra_inputs: list[Path]` (default empty), coerced to `Path` in `__post_init__`, plus read-only properties `Task.inputs` (primary + extras) and `Task.formats_in` (suffix per input). The `tasks` table gains an `extra_inputs TEXT NOT NULL DEFAULT '[]'` column with auto-migration via `ALTER TABLE ADD COLUMN` when the legacy schema is detected, so existing user databases pick up the column on next launch. `task_to_record`/`task_from_row` round-trip the field as JSON.
- `ConversionPageConfig` gains `multi_input: bool` (UI swaps the input QLineEdit for a QListWidget with Add/Remove/Clear buttons and a multi-select file dialog) and `default_options: tuple[tuple[str, object], ...]` so a page can force-inject options into every enqueued task.
- New PDF Merge sidebar page (multi_input=True, force_engine="pdf", default_options=("operation","merge")) wired to a new `PDFEngine._run_merge` operation that iterates `task.inputs` and uses `Document.insert_pdf` for each source. Requires at least two inputs and rejects encrypted PDFs.
- New Archive sidebar page powered by a Python-only `ArchiveEngine` (stdlib `zipfile`+`tarfile`, no extra binary): extract zip/tar/tgz/tbz/txz/gz/bz2/xz into a target folder. Engine rejects absolute and path-traversal entries, and uses `tarfile.extract(filter='data')` for safer member handling. Added `directory_output` flag to `ConversionPageConfig` so the output picker switches to a folder dialog and the suggested path is `<base>/<input_stem>` without an extension.
- New QR / Barcode sidebar page powered by `QREngine` wrapping `qrencode` (generate `.txt` → `.png`/`.svg` with module size, margin, and L/M/Q/H error correction) and `zbarimg` (decode any image → `.txt`, exit code 4 surfaced as a "no barcodes" error, payload written to the output file). The page sets `force_engine=True` so `png→txt` routes to QR there even though OCR remains the default for that pair globally.
- Added `extra_binaries: tuple[str, ...]` to `EngineCapabilities` so an engine can advertise secondary binaries (e.g. `zbarimg` alongside `qrencode`); `ConversionRegistry.required_binaries()` now collects both primary and extras for the dependency dialog.
- Video wave 2: GIF creator via palettegen+paletteuse filter graph (configurable fps/width with Lanczos scale), animated WebP via libwebp+loop=0 with quality 0–100, contact sheet (`select=not(mod(n,N))` + `tile=ColsxRows`) into PNG/JPG, single-frame still extraction when contact sheet is disabled, logo overlay watermark via `-filter_complex` with 9-position gravity + scale + opacity, reverse video (`reverse`+`areverse` at end of chains), and subtitle burn-in (`subtitles=` for SRT/VTT, `ass=` for ASS/SSA, with path escaping). Video page now also exposes gif/webp/png/jpg outputs and gains four new option tabs: Effects, Animation, Thumbnails, Subtitles.
- OCR wave 2: PDF input pipeline. Each PDF page renders via PyMuPDF Pixmap (300 DPI default, range 72–600), runs through tesseract per-page, and stitches into the target output — `Document.insert_pdf` for PDF, `\f` form-feed for TXT, ocr_page div concatenation with renumbered IDs for hOCR, and single-header + concatenated rows for TSV. Auto-rotate runs an OSD pre-pass (`tesseract … --psm 0`), parses `Rotate: N` (canonical 0/90/180/270 only), and re-renders via `page.set_rotation`. OCR options panel adds DPI and Auto-rotate inputs; OCR page accepts `pdf` input.
- Subtitle wave 2: ASS support. Parser handles section dispatch on `[Events]`, Format header detection (locates the `text` column index), Dialogue rows with bounded comma split (preserves commas inside the Text field), `\N`/`\n` escape sequences, and Comment skip. Formatter emits a minimal Script Info + V4+ Styles + Events document with Arial 32 white default style and `H:MM:SS.cc` time format. Subtitle page input/output formats now include `ass`.
- PDF wave 2: page reorder action (explicit page list), edit-metadata action (title/author/subject/keywords/creator), PDF→HTML extraction via PyMuPDF, and PDF repair via qpdf round-trip. PDF Tools input restricted to PDF only (previously also accepted png/jpg, duplicating the Image page).
- Audio wave 2: ID3 tag editor (title/artist/album/year/genre/track via `-metadata`) plus vocal remove via `pan=stereo|c0=c0-c1|c1=c1-c0`.
- Added a Settings sidebar page with persisted defaults at `~/.config/trex-converter/settings.json` (output folder, max concurrency, image quality, PDF DPI). Concurrency applies on next launch; output folder is honored by every conversion page when suggesting an output path.
- Added a Subtitle sidebar page powered by a new pure-Python `SubtitleEngine` (no external binary): SRT ↔ VTT conversion with optional time shift; `EngineCapabilities.requires_binary=""` is now treated as always-available by the dependency checker.
- Expanded the LibreOffice engine to a full Document format matrix: text docs ↔ DOCX/ODT/RTF/HTML/EPUB/TXT/PDF, spreadsheets ↔ XLSX/ODS/CSV/HTML/PDF, presentations ↔ PPTX/ODP/PDF (52 pairs). Document page output combo now exposes every reachable target.
- Replaced the Tesseract OCR stub with a real engine: image (png/jpg/jpeg/tif/tiff/bmp) → searchable PDF, TXT, hOCR, or TSV with multi-language picker (eng/ind/eng+ind plus custom), PSM (13 modes), and OEM (4 modes).
- Added an OCR sidebar page with a single-pane `OCROptionsPanel`; the new `force_engine` flag on `ConversionPageConfig` plus `ConversionRegistry.engine_by_name` and a `TaskQueue.engine_by_name` resolver lets the OCR page route png→pdf to Tesseract while the Image page keeps routing png→pdf to ImageMagick.
- Added an Audio sidebar page with a tabbed `AudioOptionsPanel` (Trim / Effects / Output) covering trim, fade-in/out (`afade`), gain ±20 dB (`volume`), loudness normalize (EBU R128 `loudnorm`), channel down-mix (`-ac`), and sample-rate convert (`-ar`).
- Expanded FFmpeg supported pairs to a full audio↔audio matrix (mp3/wav/aac/flac/m4a/opus/ogg) plus video→audio extraction across mp4/mov/mkv/webm sources.
- Video page now restricted to video outputs only (mp4/mov/mkv/webm); audio extraction is handled by the new Audio page to keep workflows focused.
- Expanded the FFmpeg engine with trim (start/end), resolution presets (4K/1440p/1080p/720p/480p/360p), compress (CRF + libx264 preset), rotate (90/180/270), flip H/V, free crop, speed change 0.5x–2.0x (setpts + atempo), and text watermark via drawtext with 9-position gravity and opacity.
- Added a tabbed `VideoOptionsPanel` to the Video page (Trim / Transform / Resize / Compress / Watermark).
- Expanded FFmpeg supported format pairs to a full mp4/mov/mkv/webm matrix plus video-to-audio extract; registry derives ffmpeg routing from the engine's `SUPPORTED_PAIRS`.
- Fixed PDF Tools page output-format filter that previously hid the `pdf` output, blocking every PDF→PDF operation from the UI.
- Reworked the main UI into a sidebar layout with separate Image, Video, Document, and PDF Tools pages.
- Added category-specific task forms and filtered task queues per page.
- Added real async LibreOffice document-to-PDF conversion with timeout, cancel, and output renaming.
- Added PyMuPDF-based PDF page extraction to PNG/JPG with multi-page output naming, plus PDF→TXT text extraction.
- Added PDF operations engine for extract pages (range syntax `1-3,5,8-10`), rotate, compress (garbage + deflate + clean), encrypt/decrypt with AES-256, strip metadata, and text watermark with 9-position gravity and opacity.
- Added a tabbed `PDFOperationsPanel` to the PDF Tools page (Pages / Security / Compress / Watermark / Metadata).
- Routed `pdf→pdf` and `pdf→txt` to the PyMuPDF engine; the dependency checker now supports a `python:<module>` prefix to verify Python module availability.
- Expanded the ImageMagick engine with transform (rotate, flip, flop, auto-trim, free crop, aspect crop), resize modes (dimension, longest edge, percent, megapixel), color (grayscale, sepia, negate, normalize, brightness, contrast, gamma), filter (blur, sharpen, denoise, vignette), border/frame, text watermark with gravity and opacity, output density, and ICO multi-resolution auto-resize.
- Added a tabbed `ImageOptionsPanel` to the Image page exposing all advanced ImageMagick options.
- Added `extra_options_factory` hook to `ConversionPageConfig` so any page can attach a category-specific options panel.
- Added Dashboard and About sidebar pages; each conversion page now has Convert / Queue tabs and the queue panel shows file thumbnails or mime icons.
- Added a roadmap document at `next-development.md` tracking advanced features per module and proposed new modules (Audio, Subtitle, Ebook, Archive, QR/Barcode, Metadata).
